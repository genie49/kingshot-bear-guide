# 킹샷 곰 사냥 초보자 가이드 — 디자인 스펙

날짜: 2026-07-06

## 목적

킹샷(Kingshot) 연맹 초보자들이 곰 사냥(Bear Hunt) 집결에 **잘못된 영웅**으로 참여하는 문제를 해결하는 모바일 전용 카드뉴스형 가이드 웹사이트. 1세대 기준 올바른 영웅(첸코·연우·아마네) 선택과 병력 설정 방법을 스크린샷과 함께 단계별로 안내한다.

## 대상 & 범위

- **대상**: 킹샷 1세대 초보 연맹원 (모바일에서 링크로 접속)
- **포함**: 영웅 선택 기초, 배치 화면 조작법, Equalize 병력 분배, 집결 거리 팁
- **제외**: 보상 설명, 2세대 이상 영웅, 랠리 리더용 심화 전략

## 기술 스택

- Vite + React + TypeScript
- `embla-carousel-react` — 스와이프 캐러셀 (7kb, 현재 인덱스 API 제공)
- 라이브러리 최소화, 정적 빌드 산출물 (`dist/`) — 호스팅은 사용자가 직접
- git 저장소로 관리

## UX 구조

- **모바일 전용** 세로 화면 기준. 데스크톱 접속 시 모바일 폭(최대 ~480px) 중앙 정렬
- 풀스크린 카드 1장 = 1스텝, **좌우 스와이프**로 이동 (카드뉴스 스타일)
- 하단 **진행 도트 + 카드 카운터** (예: 3/12)
- 첫 카드에서 "← 밀어서 넘기기" 힌트 애니메이션 (한 번 스와이프하면 사라짐)
- 다크 게임 테마 (곰 사냥 분위기), 큰 글씨, 짧은 문장
- 스크린샷은 카드 안에서 세로로 크게, 빨간/파란 마킹 그대로 사용 (마킹 자체가 설명)

## 카드 구성 (12장)

| # | 카드 | 이미지 | 핵심 메시지 |
|---|------|--------|------------|
| 1 | 표지 | (영웅 초상화 3종 장식) | 🐻 곰 사냥, 영웅 이렇게 고르세요 — 1세대 초보 가이드 |
| 2 | 곰 사냥이란? | 없음(텍스트/이모지) | 연맹원이 다같이 집결로 곰을 때리는 이벤트. 많이 때릴수록 좋다 |
| 3 | 핵심 규칙 | 다이어그램(첫 슬롯 강조) | 집결에 효과가 적용되는 건 **첫 번째 영웅 슬롯뿐** |
| 4 | 정답 공개 | hero-chenko / hero-yeonwoo / hero-amane | 첸코 · 연우 · 아마네. 생김새로 구분하는 법 |
| 5 | 잘못된 예 | deploy-wrong-hero (#10) | 이런 첫 슬롯 ❌ — 아무 영웅이나 넣으면 집결에 도움이 안 됨 |
| 6 | 고치기 ① | deploy-remove-hero (#9) | 잘못 들어간 영웅은 ➖ 버튼으로 빼기 |
| 7 | 고치기 ② | deploy-empty-slot (#8) | 첫 번째 ➕ 슬롯을 눌러 영웅 추가 |
| 8 | 고치기 ③ | select-heroes-ox (#7) | 선택 팝업에서 첸코·연우·아마네만 고르기 |
| 9 | 병력은 많이! | deploy-half-troops (#5) | 수동 슬라이더는 1,053명뿐 — 아깝다. 많이 보낼수록 고득점 |
| 10 | Equalize 꿀버튼 | deploy-equalize (#6) | 저울 버튼 = 보유 병력을 march queue 수만큼 균등 분배. 한 번에 3,141명, 편하게 최대 파병 |

## 컴포넌트 구조

```
src/
  App.tsx              — 캐러셀 컨테이너 + 진행 도트/카운터
  components/
    Carousel.tsx       — embla 훅 래핑, 현재 인덱스 상태 관리
    Card.tsx           — 공통 카드 레이아웃 (제목/본문/이미지 슬롯)
    ProgressDots.tsx   — 하단 도트 + 카운터
    SwipeHint.tsx      — 첫 카드 스와이프 힌트 (1회성)
  data/cards.tsx       — 카드 12장의 콘텐츠 정의 (데이터 주도)
  assets/              — 최적화된 이미지
```

- 카드 콘텐츠는 `data/cards.tsx`에 배열로 정의 → 카드 추가/수정이 데이터 수정만으로 가능
- 이미지는 원본(`raw-assets/`)에서 리사이즈·압축(폭 ~800px, 품질 조정)하여 `src/assets/`에 배치

## 에러 처리 & 엣지 케이스

- 이미지 lazy loading (첫 카드 즉시, 나머지 지연)
- 가로 모드/태블릿: 카드 최대 폭 제한으로 대응
- JS 비활성 등 예외 상황은 범위 외 (게임 유저 = 모바일 브라우저 전제)

## 테스트 & 검증

- `npm run build` 성공 + `npm run preview`로 모바일 뷰포트에서 스와이프 동작 확인
- 12장 카드 전부 렌더링, 도트 동기화, 힌트 사라짐 확인
- Lighthouse 모바일 점수 참고 수준으로 확인 (이미지 용량)

## 성공 기준

- 초보 연맹원이 링크만 받고 스스로 첫 슬롯 영웅을 첸코/연우/아마네로 바꿀 수 있다
- 모바일에서 부드러운 스와이프, 각 카드 3초 내 이해 가능한 분량
